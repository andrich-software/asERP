# CLAUDE.md — asERP.Persistence.MSSQL

Migration assembly for SQL Server. **No authored code** — only `Migrations/` (generated by `dotnet ef`) and the `.csproj` (sole difference to the sibling providers: the `Microsoft.EntityFrameworkCore.SqlServer` package).

All rules live in [src/asERP.Persistence/CLAUDE.md](../asERP.Persistence/CLAUDE.md) — read its **Migrations** section before touching anything here. Key points:

- **Never create migrations without asking the user first.**
- Never hand-edit generated migrations except for deliberate data-preservation SQL; keep the three providers' migrations in lockstep (same names, same logical content).
- Huge `UpdateData("country", ...)` seed-timestamp diffs in regenerated migrations/snapshots are expected churn, not a schema change.
